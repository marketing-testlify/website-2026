# PART 1: WHO TESTLIFY IS

## The product
Testlify is an AI-native, skills-based pre-hire assessment platform. It replaces resume screening and gut-feel hiring with verified, data-driven skill proof — before the first interview.

**In one sentence:** Testlify helps hiring teams stop guessing and start proving who can actually do the job.

## The mission
Make skills the universal language of hiring. Remove bias, speed up decisions, and give every candidate a fair shot based on what they can do — not where they went to school or how their CV reads.

## What it actually does
- Lets recruiters build role-specific assessments from a library of 3,500+ validated tests
- Adds AI video, audio, and chat interviews on top of assessments — all in one platform
- Auto-ranks candidates by score so the shortlist builds itself
- Syncs results into 100+ ATS tools natively — no middleware, no manual data entry
- Supports 16+ languages, is EEOC-defensible, SOC 2 Type II certified, ISO 27001, GDPR, CCPA compliant

# PART 3: BRAND VOICE & TONE

## Personality
**Confident challenger.** Testlify is not the biggest player — it's the one that's clearly right for the modern hiring team. Voice is direct, warm, outcome-obsessed, and occasionally sharp.

## Tone rules
- **Lead with the pain, not the product.** Start with what the recruiter feels, then show the solution.
- **Specific > general.** "93% less phone screening" not "significant time savings."
- **Active, short sentences.** No corporate passive voice.
- **Numbers anchor credibility.** Use real stats (see Part 1). Never invent.
- **"You" not "our users."** Write to the person reading.
- **No empty adjectives.** "Powerful", "robust", "cutting-edge" — delete them.

## Headline formula (for sections)
Pattern: **[Painful truth] / [Outcome] / [Proof point]**

Examples:
- ❌ "Discover our industry-leading assessment platform"
- ✅ "Stop guessing. Start hiring on proof."
- ❌ "Improve your recruitment process with our tools"
- ✅ "Virtual Gurus cut phone interview time by 93%. Here's how."

## CTA copy hierarchy
1. **Primary:** "Try for free →" or "Start free →" (no credit card required)
2. **Secondary:** "Book a demo"
3. **Tertiary:** "See how it works", "View the test library", "Read the story"

## Eyebrow formula (section labels)
Always: SHORT UPPERCASE WORD/PHRASE + coral period
Examples: `PRICING.` `HOW IT WORKS.` `CUSTOMER PROOF.` `THE CHALLENGE.` `INTEGRATIONS.`

## Typography
**Font family:** Poppins only (Google Fonts — weights 400, 500, 600, 700, 800)

```
Section eyebrow:   ~13px · weight 700 · letter-spacing .16em · UPPERCASE · colour #8A7A7D (or coral)
                   ⚠ Always followed by a coral period "." — this is a brand signature, never omit

H1 hero:           ~54px desktop · weight 800 · letter-spacing -1.4px · clamps on mobile
H2 section:        ~30–42px · weight 800 · letter-spacing -.8px to -1.3px
H3 card:           ~18–22px · weight 700
Body:              15–18px · line-height 1.55–1.64 · colour #5A4B4E
Caption / meta:    13–14px · colour #8A7A7D
```

## Spacing
```
Section vertical padding:  96px top + 96px bottom (symmetric)
Hero:                      Taller, must clear the fixed header
Content max-width:         1240px centred
Gutters:                   28px
Card border-radius:        16–20px
Tile / input radius:       12–15px
Pill / tag radius:         full (9999px)
Card shadow:               0 16px 30px rgba(110,11,14,.10)  ← warm, not cool
```

## Section background rhythm (NEVER break this)
Sections must alternate. No two adjacent sections share a background. Cycle:
**White `#FFF` → Sand `#FBF3EE` → Rose/gradient accent → White → Sand...**

If you reorder, add, or remove sections, check the rhythm and rebalance.

## Motion & interaction spec
```
Scroll-reveal:       Elements rise 28px + fade in on scroll
                     Staggered: data-delay 0.06–0.32s per element
                     Above-the-fold: reveals immediately (never gate the hero behind scroll)

Scroll progress bar: 3px coral gradient strip, fixed top, fills on scroll

Button:              Sheen/shimmer effect on hover
Card:                Subtle 3D tilt toward cursor
Globe (homepage):    Interactive draggable; toggles dark-globe ↔ light-rays ↔ dark-rays
                     Toggling must never shift layout height

Demo tabs:           Cross-fade with per-tab looping motion graphics
Testimonial:         Carousel advances exactly one slide at a time
FAQ:                 Accordion open/close
Mega-menu:           Hover/focus with full keyboard state support
```

## Component rules
```
Icons:         Inline SVG line icons only (1.8–2px stroke) — NO emoji, anywhere, ever
Images:        Load from https://testlify.com/wp-content/uploads/... (re-host in production)
Logo:          Text wordmark "Testlify." with coral dot (or swap brand/ files for production)
Responsive:    ≤900px: grids collapse to 1-col · nav → hamburger drawer · headlines clamp
               Touch targets ≥44px always
```

# PART 8: WHAT MAKES TESTLIFY DESIGN DISTINCTIVE

These are the visual signatures that set Testlify pages apart. Preserve them on every page — they compound into a coherent brand identity.

1. **Coral period after every section eyebrow.** `PROOF.` `PRICING.` `HOW IT WORKS.` Never omit.
2. **Pain-led headlines.** The customer's frustration before the product benefit.
3. **Warm, not cool.** All shadows, surfaces, and borders use warm rose/coral undertones. Never a cold grey.
4. **Scroll reveals everything.** No section appears static. Elements rise and fade on scroll, staggered.
5. **Numbers in coral.** Any big metric or proof stat uses coral `#F23F44` for the number, ink for the label.
6. **No emoji.** Anywhere. SVG icons only.
7. **Alternating backgrounds.** White → Sand → accent. Rhythm is sacred.
8. **One font.** Poppins. No exceptions.
9. **Real quotes, real names.** Never generic "happy customer" placeholders. Use the verbatim quotes from Part 1.
10. **Specific stats only.** Never "many teams" or "significant savings." Use the real numbers.

# PART 9: CREATIVE FREEDOM — WHERE TO PUSH

The design system is a floor, not a ceiling. Within the tokens and rules, you have full creative latitude on:

- **Section order** — Reorder sections to suit the page's narrative arc. Not every page needs the same flow.
- **Layout compositions** — 2-col, 3-col, asymmetric, full-bleed, overlapping cards, split-screen. Try things.
- **Motion concepts** — New animation ideas are welcome as long as they don't conflict with the existing interactions spec.
- **Visual metaphors** — The globe, pipeline, and scan beam (homepage) set the tone. New product pages can introduce new metaphors appropriate to that feature.
- **Section variants** — A "stats band" could be a horizontal strip, a 2x2 grid, a marquee scroll, or a stacked reveal. The content is fixed; the presentation is yours.
- **Illustration / iconography style** — The existing SVG icons are line-based. You can introduce more complex icon sets or custom illustrations as long as they stay on-brand (warm, clean, no cartoonish).
- **Dark sections** — The manifesto and CTA band use dark ink. You can introduce more dark-background sections where they add drama. Just maintain the alternating rhythm.
- **Micro-interactions** — New hover states, progress indicators, interactive elements are encouraged. Match the feel (smooth, warm, purposeful — not flashy for its own sake).

**The creative test:** Would a TA Manager at a 500-person company feel like this was built for them? If yes, ship it.
